Error: $http:baddata
Bad JSON Data

Data must be a valid JSON object. Received: "{0}". Parse error: "{1}"

Description

The default @{link ng.$http#default-transformations transformResponse} will try to parse the response as JSON if the Content-Type header is application/json or the response looks like a valid JSON-stringified object or array. This error occurs when that data is not a valid JSON object.

The error message should provide additional context such as the actual response.

To resolve this error, make sure you pass valid JSON data to transformResponse or use an appropriate Content-Type header for non-JSON data.
